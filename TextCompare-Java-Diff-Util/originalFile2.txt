<h3 class='post-title entry-title' itemprop='name'>
C does not support call by reference
</h3>
<div class='post-header'>
<div class='post-header-line-1'></div>
</div>
<div class='post-body entry-content' id='post-body-8911441976864498858' itemprop='description articleBody'>
<div dir="ltr" style="text-align: left;" trbidi="on">
<div style="text-align: justify;">
<span style="color: #990000;"><i><b>C does not support call by reference.</b></i></span> Because when you pass pointer then you are passing the address of the variable which is actually a value.</div>
<div style="text-align: justify;">
<br /></div>
<div style="text-align: justify;">
If a language is pass by reference, then there are no different ways of passing the argument &#8211; you pass an argument and a reference is sent to the function &#8211; there are no syntactical differences between passing by value and passing by reference.&nbsp;</div>
<div style="text-align: justify;">
<br /></div>
<div style="text-align: justify;">
However, in C, when the intent is to pass by reference, then we change the syntax a bit &#8211; that is we don't pass the variable anymore, but pass the address of it. Technically speaking, the address is still _copied_ to the parameter of the function and we use that _copied_ address to dereference it and get the value stored at the address.&nbsp;</div>
<div style="text-align: justify;">
<br /></div>
<div style="text-align: justify;">
So, pointers is a tool to simulate/emulate call by reference, but the language as such does not _naturally_ support call by reference. So, C is strictly call by value and not by reference.</div>
<div style="text-align: justify;">
<br /></div>
</div>
